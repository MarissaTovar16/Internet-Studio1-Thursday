What is the difference between structured content and semantic HTML?

  • Structured content is planned and organized. It is content legiblly constructed outside for the interface. Semantic HTML is the elements within the language that communicates the information.

How do they work together?

  • The semantic elements make up the structured content. But, not so much for stylistic purposes. Rather, it serves as organized elements in a system that translates it to a visual/legible site.

Why is it important to have structured content and semantic HTML in your web pages?

  • It is important to be able to easily read your code. Not only does it help legibility of the site, but if done well, can also assist algorithms.
